Overview

The User Scenario through User Interaction Diagrams (US-UIDs) are a format of automated acceptance tests. They are created by Web browser and executed by Eclipse Plug-in. The Figure 1 shows the framework architecture.


Figure 1. Framework architecture.

The framework is composed by:

  • Web editor - it is used by customers/user to create US-UIDs;
  • Eclipse plug-in - it is user by developers to tests automation.

Web editor

At http://agil.inf.ufsc.br/us is possible create an user account. After that the user is created, it is possible to create projects and in each project create the US-UIDs. The Figure 2.0 shows the authentication page.


Figure 2.0 - Authentication page.

The Figure 2.1 shows the page to create projects. The projects are a way of organizing scenarios.


Figure 2.1 - Projects page.

Within each project the US-UIDs files are created. The Figure 2.2 shows the page to create US-UIDs. The US-UIDs need to have and package such as 'net.douglas.ufsc' and a name such as 'Calculator'.


Figure 2.2 - Scenarios page.

The Figure 2.3 shows the editor of US-UIDs.


Figure 2.3 - Sc3n4r10 editor.

Eclipse plug-in

Installation

  • Menu: Eclipse -> Help -> Install New Software
  • Add Repository
    • Name: Scenario
    • Location: http://agil.inf.ufsc.br/us/site-plugin/

Creating a java project with US-UIDs

Inside of Java project, the dependency us.jar need be installed. The Figure 3.0 shows a Java project with the dependency.

Figure 3.0 - Java project.

Downloading of scenarios from Web

Inside of Java project create a folder, such as 'scenario'. To download the scenarios:

  • Select the folder;
  • Click with mouse right button;
  • Click in download of US-UIDs;
  • Fill the server configuration.
The Figure 3.1 shows a server configuration.

Figure 3.1 - Server configuration to download.

The plug-in Scenario is composed by three view: Test, Fixture name and Scenario. It is possible search 'scenario' and open the view using the 'Quick Access' in eclipse. The Figure 3.2 shows the Scenario views.


Figure 3.2 - Scenario views.

Creating glue code

  • Select project (click in root path);
  • Click with mouse right button;
  • Click in Scenario "Generate glue code from Us-UIDs".

Figure 3.3 shows filling the package to generate glue code template.


Figure 3.3 - filling the package to generate glue code.

Figure 3.4 shows the glue code template generated. The glue code need be filled. It should be linked the glue code with the application.


Figure 3.3 - Glue code template.

Running the scenario

  • Select the scenario from download folder;
  • Click with mouse right button;
  • Select "Run as";
  • Click in "Scenario".

Figure 3.5 shows the Eclipse plug-in running the example of calculator scenario.
